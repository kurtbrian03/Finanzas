[1;34m============================================================[0m
[1;36m1. VISIÓN GENERAL DEL SISTEMA (OVERVIEW)[0m
[1;34m============================================================[0m

[0;37mEste sistema es una aplicación modular orientada al análisis de documentos
(PDF, Excel), validación fiscal (RFC, folios), descargas controladas y
gestión de historial, construida sobre una arquitectura multiarchivo
profesional. La lógica se encuentra organizada por capas y responsabilidades,
evitando el monolito y favoreciendo la escalabilidad y el mantenimiento.[0m

Objetivos clave del sistema:
- Proveer una interfaz clara para cargar y analizar documentos.
- Validar información fiscal de forma estructurada y extensible.
- Permitir descargas manuales, nunca automáticas.
- Mantener un historial de acciones para auditoría.
- Separar UI, lógica de negocio, validación, descargas y configuración.

Este documento describe la arquitectura, la estructura de carpetas, los
flujos de ejecución, los diagramas de alto nivel, el roadmap de evolución
y un manual técnico para desarrolladores.


=====================================================================
2. ARQUITECTURA GENERAL DEL SISTEMA (ISO 2.x)
=====================================================================

2.1 Descripción General de la Arquitectura
------------------------------------------
La arquitectura sigue un enfoque por capas y módulos, donde cada carpeta
representa una responsabilidad bien definida. El objetivo es:

- Reducir el acoplamiento entre componentes.
- Aumentar la cohesión dentro de cada módulo.
- Facilitar pruebas, refactor y extensiones futuras.
- Mantener un punto de entrada mínimo (app.py) que delega todo.

2.2 Capas Principales
---------------------
- UI Layer  
  - Responsabilidad: Interfaz de usuario, layout, navegación, componentes visuales.
  - Interactúa con: core.router, core.state_manager.
- Core Layer  
  - Responsabilidad: Enrutamiento, manejo de estado, ciclo de vida, eventos internos.
  - Interactúa con: todas las demás capas.
- Analysis Layer  
  - Responsabilidad: Análisis de PDFs y Excels, extracción de tablas, detección de entidades.
- Validation Layer  
  - Responsabilidad: Validación de RFC, folios, reglas SAT, homoclave.
- Downloads Layer  
  - Responsabilidad: Descargas manuales, empaquetado, filtros de descarga.
- History Layer  
  - Responsabilidad: Historial de acciones, auditoría, persistencia de eventos.
- Config Layer  
  - Responsabilidad: Configuración, constantes, entorno (dev/prod).
- Utils Layer  
  - Responsabilidad: Funciones auxiliares reutilizables (archivos, errores, formatos).

2.3 Flujo de Datos Global
-------------------------
1. El usuario interactúa con la UI (carga un archivo, ingresa un RFC, solicita una descarga).
2. La UI envía la intención al router (core.router).
3. El router decide qué módulo debe manejar la acción (analysis, validation, downloads, etc.).
4. El módulo ejecuta la lógica correspondiente (análisis, validación, empaquetado).
5. El state_manager (core.state_manager) guarda los resultados y el estado actual.
6. La UI lee el estado y muestra resultados, mensajes y opciones de acción.
7. El history_manager registra eventos relevantes (acciones del usuario, descargas, validaciones).

3. ESTRUCTURA COMPLETA DEL PROYECTO (ISO 3.x)
=====================================================================

3.1 Estructura de Carpetas y Responsabilidades
----------------------------------------------

/core  
    state_manager.py        # Manejo centralizado de estado (session_state, flags, resultados).  
    router.py               # Enrutamiento de vistas y acciones de usuario.  
    event_bus.py            # Sistema de eventos internos entre módulos (pub/sub simple).  
    lifecycle.py            # Hooks de ciclo de vida (inicio, cierre, reinicio lógico).  

/ui  
    layout.py               # Layout principal: panel izquierdo, central, derecho, header, footer.  
    components.py           # Componentes reutilizables: tablas, cards, alerts, loaders, formularios.  
    navigation.py           # Lógica de navegación: rutas, secciones, selección de vista.  
    theme.py                # Definición de colores, tipografías, tamaños, estilos visuales.  
    messages.py             # Textos, labels, mensajes de error/éxito, tooltips, textos de ayuda.  

/analysis  
    pdf_analyzer.py         # Lógica de análisis de PDFs: texto, metadatos, estructura.  
    excel_analyzer.py       # Lógica de análisis de Excel: hojas, columnas, tipos, estadísticas.  
    table_extractor.py      # Extracción de tablas desde PDF/Excel.  
    entity_detector.py      # Detección de entidades (RFC, montos, fechas, folios, etc.).  
    tone_analyzer.py        # Análisis de tono o clasificación básica de contenido (opcional).  

/validation  
    rfc_validator.py        # Validación estructural de RFC (longitud, patrón, homoclave).  
    folio_validator.py      # Validación de folios fiscales (formato, longitud, patrón).  
    sat_rules.py            # Reglas de negocio alineadas a SAT (estructura, patrones, restricciones).  
    homoclave_engine.py     # Cálculo y validación de homoclave (si aplica).  

/downloads  
    downloader.py           # Lógica de generación de descargas manuales (botones, streams).  
    download_filters.py     # Filtros para decidir qué se puede descargar y en qué formato.  
    file_packager.py        # Empaquetado de archivos (zip, csv, combinaciones).  
    preview_metadata.py     # Metadatos de archivos para vista previa y descargas (nombre, tipo, tamaño).  

/history  
    history_manager.py      # Manejo de historial en memoria o persistente (acciones del usuario).  
    audit_log.py            # Registro de acciones relevantes para auditoría.  
    persistence.py          # Abstracción de persistencia (archivo local, DB futura, etc.).  

/config  
    settings.py             # Configuración general (flags, parámetros, límites).  
    constants.py            # Constantes globales (nombres de columnas, claves, rutas lógicas).  
    environment.py          # Variables de entorno, modos dev/prod, configuración externa.  

/utils  
    file_utils.py           # Utilidades de archivos (lectura, validación de extensión, paths).  
    error_utils.py          # Manejo y formateo de errores, trazas amigables.  
    format_utils.py         # Formateo de textos, números, fechas, máscaras.  
    pdf_utils.py            # Utilidades específicas para PDF (lectura, extracción).  
    excel_utils.py          # Utilidades específicas para Excel (lectura, validación).  

app.py                      # Punto de entrada mínimo: configura Streamlit, inicializa router y layout.

4.3 Flujo del Router (Detallado)
--------------------------------

El usuario realiza una acción en la UI (ej. seleccionar "Análisis PDF").

ui.navigation determina la ruta lógica (ej. "analysis_pdf").

ui.layout llama a core.router con la acción seleccionada.

core.router evalúa la acción y decide qué módulo ejecutar:

"analysis_pdf" → analysis.pdf_analyzer

"analysis_excel" → analysis.excel_analyzer

"validate_rfc" → validation.rfc_validator

"download_results" → downloads.downloader

El módulo ejecuta la lógica correspondiente.

core.state_manager guarda los resultados (ej. análisis, errores, flags).

ui.layout y ui.components leen el estado y renderizan la vista actualizada.

Código

4.4 Flujo de Análisis PDF (Detallado)
-------------------------------------

Usuario carga un archivo PDF desde la UI.

ui.components llama a utils.file_utils para validar extensión y tamaño.

Si es válido, se envía el archivo al router con la acción "analysis_pdf".

core.router llama a analysis.pdf_analyzer con el archivo.

pdf_analyzer:

Extrae texto y metadatos.

Llama a table_extractor para detectar tablas.

Llama a entity_detector para encontrar RFC, montos, fechas, etc.

Opcionalmente llama a tone_analyzer para clasificar el contenido.

Los resultados se guardan en core.state_manager.

ui.layout muestra:

Vista previa del PDF (solo lectura).

Resultados de análisis (tablas, entidades, métricas).

history_manager registra que se realizó un análisis PDF.

Código

4.5 Flujo de Validación Fiscal (RFC/Folio)
------------------------------------------

Usuario ingresa un RFC o folio fiscal en la UI.

ui.components envía la acción "validate_rfc" o "validate_folio" al router.

core.router llama a:

validation.rfc_validator para RFC.

validation.folio_validator para folios.

rfc_validator:

Usa sat_rules para validar estructura, longitud y patrón.

Usa homoclave_engine para validar homoclave (si aplica).

El resultado (válido/ inválido, motivos) se guarda en state_manager.

ui.layout muestra el resultado con mensajes claros (válido, inválido, por qué).

history_manager registra la validación realizada.

Código

4.6 Flujo de Descargas (Manual y Controlado)
--------------------------------------------

Usuario selecciona una opción de descarga (ej. "Descargar resultados en CSV").

ui.components envía la acción "download_results" al router.

core.router llama a downloads.downloader.

downloads.downloader:

Usa download_filters para verificar qué se puede descargar.

Usa file_packager para generar el archivo final (CSV, ZIP, etc.).

La UI muestra un botón de descarga manual (nunca automática).

El usuario hace clic y se dispara la descarga.

history_manager registra la descarga (tipo, fecha, contexto).

Código
